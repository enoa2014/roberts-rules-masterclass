# 12-前端详细设计

> 状态：Draft
> 基于 `ui-ux-pro-max` 智能化建议与 `docs/01`~`docs/11` 需求文档产出。

## 1. 设计概览 (Design System)

### 1.1 核心理念

- **风格关键词**: Professional (专业), Academic (学术), Clarity (清晰), Trust (信任).
- **用户画像**: 中小学教师、家长（非低龄儿童）。需要高效、清晰的操作界面。
- **响应式策略**: Mobile-First (移动端优先)，兼容 375px (微信) 与 1440px (桌面)。

### 1.2 色彩体系 (Color Palette)

| 角色 | 色值 | 说明 |
|------|------|------|
| **Primary** | `#1E40AF` (Blue-800) | 主品牌色，用于导航、重点按钮。传递专业与信任。 |
| **Secondary** | `#3B82F6` (Blue-500) | 次级操作、链接色。 |
| **Accent** | `#0891B2` (Cyan-600) | 强调色，用于高亮信息。 |
| **Success** | `#10B981` (Emerald-500) | 成功状态、正向反馈。 |
| **Warning** | `#F59E0B` (Amber-500) | 警告、注意。 |
| **Error** | `#EF4444` (Red-500) | 错误、危险操作。 |
| **Surface** | `#FFFFFF` | 卡片与内容背景。 |
| **Background** | `#F8FAFC` (Slate-50) | 页面整体背景，降低视觉疲劳。 |
| **Text** | `#0F172A` (Slate-900) |这里正文。 |
| **Muted** | `#64748B` (Slate-500) | 辅助文字。 |

### 1.3 字体排版 (Typography)

> 采用系统默认字体栈，确保加载速度与多端一致性。

- **Font Family**: `Inter`, `system-ui`, `-apple-system`, `sans-serif`.
- **Headings**:
  - H1: `text-2xl` / `text-4xl` (Mobile/Desktop), `font-bold`, `tracking-tight`.
  - H2: `text-xl` / `text-2xl`, `font-semibold`.
  - H3: `text-lg` / `text-xl`, `font-semibold`.
- **Body**:
  - Base: `text-base` (16px), `leading-relaxed`.
  - Small: `text-sm` (14px), `text-muted`.

### 1.4 交互与动效 (Interaction & Motion)

- **Hover**: 统一使用 `transition-colors duration-200`。
- **Feedback**: 点击操作必须有视觉反馈（颜色变化或涟漪）。
- **Loading**: 按钮内嵌 Spinner，页面级使用 Skeleton 骨架屏。
- **Toast**: 右上角（桌面）或 顶部居中（移动端）弹出提示。

## 2. 组件规范 (Component Library)

### 2.1 按钮 (Buttons)

- **Primary**: `bg-primary text-white hover:bg-primary/90 shadow-sm`.
- **Secondary**: `bg-white text-primary border border-gray-200 hover:bg-gray-50`.
- **Ghost**: `text-gray-600 hover:text-gray-900 hover:bg-gray-100`.
- **Destructive**: `bg-red-500 text-white hover:bg-red-600`.
- **Sizes**:
  - Default: `h-10 px-4 py-2`
  - Sm: `h-9 px-3`
  - Lg: `h-11 px-8`

### 2.2 表单 (Forms)

- **Input**: `border-gray-200 focus:ring-2 focus:ring-primary/20 rounded-md`.
- **Label**: `text-sm font-medium text-gray-700 mb-1 block`.
- **Validation**: 错误信息显示在输入框下方，使用 `text-red-500 text-sm`.

### 2.3 卡片 (Cards)

- **Style**: `bg-white rounded-xl shadow-sm border border-gray-100/50`.
- **Hover**: `hover:shadow-md transition-shadow`.

### 2.4 导航 (Navigation)

- **Desktop**: 顶部固定 Header，Logo 左侧，菜单右侧。
- **Mobile**: 底部 TabBar (学员端) 或 汉堡菜单 (通用)。

## 3. 页面详细设计

### 3.1 登录/注册 (Auth)

- **Layout**: 居中卡片布局 (`min-h-screen flex items-center justify-center bg-gray-50`).
- **Components**:
  - Logo 居中展示。
  - 表单区域：用户名、密码。
  - 底部链接：“注册账号” / “返回首页”。
- **UX**:
  - 登录成功：Toast "登录成功"，延迟 跳转。
  - 登录失败：表单整体抖动或顶部 Error Alert。

### 3.2 邀请码页 (Invite)

- **Layout**: 重点突出的单页布局。
- **Components**:
  - 巨大的输入框 (仿 Google 搜索框风格)。
  - "验证并加入" 按钮。
  - 下方说明文字：解释如何获取邀请码。
- **Interaction**:
  - 输入时自动大写。
  - 提交后 Loading 状态。
  - 成功后显示全屏 Success 动画，然后跳转首页/课程页。

### 3.3 课程首页 (Dashboard)

- **Mobile View**:
  - **Banner**: 课程价值主张，配图。
  - **Grid**: 2x2 网格导航 (议事规则、阅读探究、工具箱、资源)。
  - **List**: 最近动态/通知。
- **Desktop View**:
  - 侧边栏导航 + 顶部 Header。
  - 内容区域宽屏展示，卡片横向排列。

### 3.4 互动课堂 (Classroom)

> 核心高频页面，需极简与防误触。

- **Teacher View**:
  - **控制台**: 底部固定操作栏 (开始/结束，发起投票，点名)。
  - **主视图**: 学员列表/状态墙 (Grid 布局)。
  - **举手队列**: 侧边抽屉或浮层，实时显示举手学生。
- **Student View**:
  - **状态栏**: 顶部显示当前环节 (如：自由讨论、投票中)。
  - **互动区**: 居中大按钮 (举手/取消举手)。
  - **投票弹窗**: 模态框 (Modal) 强制打断，必须提交才能关闭。

### 3.5 阅读站 (Reading)

- **Wrapper**: 全屏布局，移除多余 Header/Footer，仅保留左上角“返回”悬浮钮。
- **Content**: `iframe` 嵌入，`height: 100vh`。

## 4. 实施 Checklist (Pre-Delivery)

基于 `ui-ux-pro-max` 建议的验收清单：

- [ ] **Icons**: 使用 SVG (Lucide-React)，拒绝 Emoji 图标。
- [ ] **Cursor**: 所有可点击区域必须 `cursor-pointer`。
- [ ] **Whitespace**: 保持充足留白，避免密集排版。
- [ ] **Feedback**: 按钮点击后的 Loading 态必须实现，防止重复提交。
- [ ] **Empty States**: 列表为空时显示插画/引导文案。
- [ ] **Mobile**: 确保无横向滚动条，点击区域 (Touch Target) > 44px。
- [ ] **Dark Mode**: MVP 暂不强制适配深色模式，但在代码中预留 class 策略 (强制 light mode)。

---

## 5. CSS 架构建议 (Next.js + Tailwind)

- `globals.css`: 定义 `@layer base` 基础样式 (Headings, Body)。
- `tailwind.config.ts`: 扩展 `colors`、`fontFamily`。
- 组件级样式：尽量使用 Utility Classes，复杂动效使用 `framer-motion`。
