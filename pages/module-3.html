模块三：动议 (Motions) 的精密机械

如果是"老友十二条"是步枪，那么"动议体系"就是重机枪。掌握了它，你就能在任何复杂会议中掌控节奏。

3.1 动议的生命周期

一个想法要变成集体的决议，必须经历标准流程：

1. 动议
→
2. 附议
→
3. 陈述
→
4. 辩论
→
5. 表决
→
6. 宣布

关键点：

  • 附议 (Second)："我附议"并不代表我同意这个观点，只代表"我同意我们将时间花在讨论这个问题上"。这是为了过滤掉只有一个人感兴趣的琐事。
  • 陈述 (State the Question)：如果不经主持人重复陈述，动议就不算正式开始。这是主持人"接球"的动作。

3.2 动议优先体系 (Rank of Motions)

这是罗伯特规则中最难也最精彩的部分。当多个动议同时出现时，谁能打断谁？谁必须先处理？

优先动议 (Privileged) 级别最高 · 无需附议 · 不可辩论
休会 / 权益问题 / 提议议程
附属动议 (Subsidiary) 附着于主议题 · 协助处理
搁置 / 停止辩论 / 延期 / 付委 / 修改
主动议 (Main Motion) 级别最低 · 只有空闲时才能提
引入新议题

核心逻辑：LIFO (后进先出)

会议处理动议的顺序就像"叠盘子"。如果有更高级别的动议提出，它就叠在当前议题之上，必须先处理上面的盘子 (Pending)，才能处理下面的 (Immediately Pending)。

案例演示：
1. 主动议： "我们周末去爬山。" (正在讨论中...)
2. 附属动议 (修改)： A提议 "把地点改成海边"。(打断了爬山，现在讨论海边)
3. 优先动议 (休会)： B突然喊 "太晚了，我动议休会"。(打断了海边)

处理顺序： 先表决B的休会 -> (假设失败) -> 回来讨论A的海边 -> (假设通过) -> 最后讨论"去海边"的主动议。

3.3 修正案的修正案

罗伯特规则只允许两级修改：

  1. 一级修正： 修改主词。 "去爬山" -> "去海边"。
  2. 二级修正： 修改一级修正案。 "去海边" -> "去黄金海岸"。

禁止三级修正： 如果有人想把"黄金海岸"改成"钻石海岸"，必须先等二级修正表决完，才能再提新的。
